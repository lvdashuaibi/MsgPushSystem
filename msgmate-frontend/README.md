# MsgMate 消息推送系统前端

基于 Vue 3 + TypeScript + Element Plus 构建的现代化消息推送系统前端界面。

## 🚀 功能特性

### ✅ 已完成功能

#### 📧 消息管理
- **消息发送** - 支持立即发送和定时发送
  - 选择消息模板
  - 填写模板变量
  - 设置接收者
  - 选择发送时间
- **消息模板管理** - 完整的CRUD操作
  - 创建/编辑/删除模板
  - 模板预览
  - 多渠道支持（邮件、短信、飞书、微信、钉钉）
- **消息记录查询** - 消息发送历史
  - 按消息ID快速查询
  - 消息详情查看
  - 发送状态跟踪

#### 👥 用户管理
- **用户列表管理** - 完整的用户CRUD
  - 创建/编辑/删除用户
  - 用户状态管理
  - 多联系方式支持
- **标签管理** - 用户分组和标签
  - 标签统计分析
  - 标签云图展示
  - 按标签查找用户
  - 批量消息发送

#### ⏰ 定时消息
- **定时消息列表** - 查看所有定时任务
  - 任务状态管理
  - 取消定时任务
  - 执行历史查看
- **创建定时消息** - 灵活的定时发送
  - 指定用户或按标签选择
  - 模板选择和数据填充
  - 时间选择和快捷设置
  - 草稿保存功能

#### 📊 仪表盘
- **数据统计** - 系统概览
  - 消息发送统计
  - 用户数量统计
  - 模板使用情况
  - 定时任务状态

## 🛠️ 技术栈

- **框架**: Vue 3 + TypeScript
- **UI库**: Element Plus
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP客户端**: Axios
- **时间处理**: Day.js
- **构建工具**: Vite
- **代码规范**: ESLint + Prettier

## 📦 安装和运行

### 环境要求
- Node.js >= 16
- npm >= 8

### 安装依赖
```bash
npm install
```

### 开发环境运行
```bash
npm run dev
```

### 构建生产版本
```bash
npm run build
```

### 预览生产版本
```bash
npm run preview
```

## 🔧 配置说明

### API代理配置
项目已配置API代理，开发环境下会自动将 `/api` 请求代理到后端服务 `http://localhost:8109`。

### 环境变量
可以通过 `.env` 文件配置环境变量：
```env
VITE_API_BASE_URL=http://localhost:8109
```

## 🎯 核心功能说明

### 消息发送流程
1. 选择或创建消息模板
2. 填写模板变量数据
3. 选择接收者（用户ID或标签）
4. 设置发送时间（立即或定时）
5. 确认发送

### 用户标签系统
- 支持为用户添加多个标签
- 标签统计和可视化展示
- 按标签批量选择用户
- 标签云图直观展示热门标签

### 定时消息调度
- 支持按用户ID或标签创建定时任务
- 灵活的时间选择（快捷时间 + 自定义）
- 任务状态实时跟踪
- 支持取消未执行的任务

## 🔗 与后端集成

前端已完整实现与后端API的对接：

- **消息接口**: `/msg/*`
- **用户接口**: `/user/*`
- **定时消息接口**: `/scheduled/*`

所有接口都包含完整的错误处理和加载状态管理。

## 📝 开发说明

### 添加新功能
1. 在 `src/api/` 中定义API接口
2. 在 `src/types/` 中定义TypeScript类型
3. 在 `src/views/` 中创建页面组件
4. 在 `src/router/` 中添加路由配置

项目使用ESLint和Prettier进行代码规范检查：

npm run lint
